# Guía de contribución

Gracias por tu interés en este repositorio. Si bien es principalmente un portafolio académico personal, se aceptan sugerencias, correcciones y mejoras.

---

## ¿Cómo contribuir?

### 1. Reportar errores o sugerencias

Si encontrás un error en el código, la documentación, o tenés una sugerencia de mejora:

1. Abrí un **Issue** en GitHub describiendo el problema o la propuesta.
2. Incluí el contexto necesario: qué materia, qué archivo, qué comportamiento esperabas y qué observaste.

### 2. Proponer cambios mediante Pull Request

1. Hacé un **fork** del repositorio.
2. Creá una rama con un nombre descriptivo:
   ```bash
   git checkout -b fix/descripcion-del-cambio
   ```
3. Realizá los cambios respetando las [convenciones de nombres](docs/convenciones-de-nombres.md).
4. Asegurate de que el código compila y/o funciona correctamente antes de enviar.
5. Abrí un **Pull Request** con una descripción clara de los cambios realizados.

---

## Reglas básicas

- Respetar la estructura de carpetas existente.
- Seguir las convenciones de nombres definidas en [`docs/convenciones-de-nombres.md`](docs/convenciones-de-nombres.md).
- No subir archivos binarios, compilados, ni dependencias al repositorio.
- Usar mensajes de commit claros y en español o inglés.
- Mantener un tono profesional y constructivo.

---

## Código de conducta

Este repositorio sigue principios de respeto y colaboración. No se toleran comentarios ofensivos, discriminatorios o malintencionados.

---
